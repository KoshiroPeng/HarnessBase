# 前端设计评审清单

## 评审重点

1. 交互设计
2. 接口调用设计
3. 前端业务逻辑设计

## 清单

| 重要性 | 序号 | Checklist |
| --- | --- | --- |
| 必选 | 1 | 是否按模板要求编写详细设计说明书？ |
| 必选 | 2 | 是否有路由设计，是否合理？ |
| 必选 | 3 | 路由命名是否规范，是否与现有路由冲突？ |
| 必选 | 4 | 是否有出入口设计，是否合理？ |
| 必选 | 5 | 是否有组件设计，是否可复用已有组件？ |
| 必选 | 6 | 是否有交互设计，是否符合业务场景？ |
| 必选 | 7 | 表单交互设计是否体现验证条件？ |
| 必选 | 8 | 是否考虑异常长度、敏感操作确认等保护？ |
| 必选 | 9 | 是否有调用接口说明，是否体现关键参数和异常提示？ |
| 必选 | 10 | 前端是否有业务处理逻辑，是否合理？ |
| 必选 | 11 | 接口返回数据使用是否方便，是否存在一个操作调用多个接口的情况？ |
| 必选 | 12 | 若存在前端缓存，是否有保证数据一致性的设计？ |
| 必选 | 13 | 若使用外部接口，是否有参数保护与异常提示设计？ |
| 必选 | 14 | 若复用已有方法，是否评估了对原有功能的影响？ |

## 当前项目适配说明

- 当前仓库是 Vue 2 前端应用，本清单按 [web/src/api](../../web/src/api)、[web/src/views](../../web/src/views)、[web/src/router](../../web/src/router)、[web/src/store](../../web/src/store) 的真实结构执行。
- 前端设计应优先围绕 `system`、`monitor`、`tool` 等现有功能域收敛，并先核对 [docs/design/feature-admin-domains.md](../design/feature-admin-domains.md)。
- 如果设计讨论中引用 Harness Engineering，应优先确认它是在帮助前端主链路交付，而不是引入超前平台化抽象。
