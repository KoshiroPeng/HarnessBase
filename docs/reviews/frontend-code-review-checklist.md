# 前端代码评审清单

## 评审重点

1. 代码规范
2. 程序结构合理性
3. 逻辑正确性与安全性

## 清单

| 重要性 | 序号 | Checklist |
| --- | --- | --- |
| 必选 | 1 | 变量、常量、方法命名是否符合规范？ |
| 必选 | 2 | 类、方法和关键逻辑是否有适当注释？ |
| 必选 | 3 | 是否对表单内容和接口返回做了合理校验？ |
| 必选 | 4 | 是否对异常逻辑做了合理处理，界面提示是否合理？ |
| 必选 | 5 | 循环是否一定能结束，判断条件是否合理？ |
| 必选 | 6 | 定时器是否一定能停止，是否存在频繁接口请求？ |
| 必选 | 7 | 是否存在单个方法过于繁琐，是否可提取公共逻辑？ |
| 必选 | 8 | 若使用第三方工具类，是否评估过稳定性问题？ |
| 必选 | 9 | 接口调用是否合理，是否减少了不必要请求？ |
| 必选 | 10 | 是否评估过接口返回数据量和 DOM 性能问题？ |
| 必选 | 11 | 若调整公共组件，是否评估过影响范围？ |
| 必选 | 12 | 事件监听是否有销毁，是否会影响其他组件？ |
| 必选 | 13 | `web/src/api` 中新增或保留的接口封装，是否能在后端 Controller 或运行时文档中找到对应入口？ |

## 当前项目适配说明

- 当前仓库是 Vue 2、JavaScript、Vue CLI、Element UI、Vuex，不应再按 Vue 3 / Vite / Pinia 规则评审。
- 若改动涉及 `web/src/api`、`web/src/views`、路由、store 或公共组件，必须评估影响范围并同步更新相关设计文档。
- 若发现前端 API 与后端 Controller 不一致，应先记录到 [docs/reference/README.md](../reference/README.md) 的已知差异，而不是虚构后端接口。
