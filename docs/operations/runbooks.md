# 运行手册

## 目标

本文档沉淀 HernessDemo 在发布、故障处理和恢复场景中的标准操作步骤，降低关键操作对个人经验的依赖。

## 建议手册清单

当前建议至少维护以下运行手册：

- 发布手册。
- 回滚手册。
- 数据库迁移异常处理手册。
- 应用启动失败排查手册。
- 指标异常与告警排查手册。
- 外部依赖不可用降级手册。
- 远端主机初始化手册。

## 发布手册最小内容

- 发布版本号。
- 影响范围。
- 前置检查项。
- 部署步骤。
- 验证步骤。
- 观察窗口。
- 回滚条件。
- 联系人和值班人。

## 回滚手册最小内容

- 适用场景。
- 回滚目标版本确认方式。
- 应用回滚步骤。
- 配置回滚步骤。
- 数据库兼容性检查。
- 回滚后验证步骤。

## 故障排查原则

- 先确认影响面，再执行风险动作。
- 优先依赖监控、日志和可追溯变更记录定位问题。
- 不在生产直接试错式修改。
- 紧急处置结束后必须补齐复盘。

## 当前最小落地建议

当前仓库处于早期阶段，当前已优先落地以下运行手册：

1. `发布与回滚手册`
2. `数据库迁移异常处理手册`
3. `远端主机初始化手册`

后续随着外部依赖、认证、计费和搜索能力落地，再按模块细化。

当前仓库已新增：

- `docs/operations/remote-host-bootstrap.md`
- `docs/operations/rollback-runbook.md`

用于规范首次接入新服务器、异常回滚等高风险动作。

## 维护规则

- 新增高风险变更路径时，必须补充对应运行手册。
- 发布后若出现未覆盖故障场景，必须在复盘后补充本文档。
