# 服务级目标与发布门禁

## 目标

本文档用于定义 HernessDemo 的基础 SLI / SLO 和发布门禁规则，为后续自动化验证和渐进式发布提供统一判定标准。

## 推荐 SLI

建议优先关注以下服务指标：

- 可用性。
- 请求错误率。
- 响应时间。
- 数据库连接池健康度。
- 核心业务探针成功率。

## 推荐 SLO 占位

当前项目仍处于早期阶段，建议先定义占位目标，后续基于真实流量修正：

- 核心 API 可用性目标。
- 登录与项目查询接口成功率目标。
- 关键接口 P95 响应时间目标。

## 发布门禁建议

进入下一个环境前，至少满足：

- 构建成功。
- 测试通过。
- 架构规则检查通过。
- 静态检查通过。
- 关键指标未超阈值。
- 发布后验证通过。

## 当前状态

当前仓库已经具备部分门禁基础能力，但尚未把服务目标与观测阈值固化为标准文档。本文档先作为治理占位，待真实业务上线后再补充具体阈值。

## 维护规则

- 当系统进入真实生产使用阶段后，必须把占位目标替换为量化阈值。
- 发布门禁规则变化时，必须同步更新流水线和发布验证文档。
